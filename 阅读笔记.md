# SliceGPT 代码阅读笔记

> **论文**: SliceGPT: Compress Large Language Models by Deleting Rows and Columns (ICLR 2024)
> **仓库**: https://github.com/microsoft/TransformerCompression

## 1. 核心思想

SliceGPT 通过**正交变换（旋转）**将隐藏状态映射到新基底，使信号能量集中在前几个主成分上，然后直接**截断低能量维度**。模型的 hidden dimension 物理上变小（如 4096→3072），权重矩阵变为更小的**稠密**矩阵。

关键数学性质：正交变换 $Q$ 对 RMS 归一化是不变的，通过将 $Q$ 吸收到权重中，变换不改变模型输出。

## 3. 工作流程

1. **替换层** → `replace_layers()`: 将 DecoderLayer 替换为 CompressedLayer（含 shortcut 矩阵）
2. **融合 LayerNorm** → `fuse_modules()`: 将 LN 的 γ 吸收进 Linear 权重，LN 替换为无参数 RMSN
3. **旋转 + 切片** → `rotate_and_slice()`: 逐层 PCA → 正交变换 → 维度截断
4. **评估 PPL**
5. **（可选）LoRA 恢复微调**

## 4. 旋转与切片机制

`rotate_and_slice_sequential()` 逐层交错执行：

1. `pca_calc(X)`: 计算隐藏状态协方差矩阵 $H = \sum X^T X$，特征分解得 $Q$（按特征值降序）
2. 旋转 Embedding: $W_{emb} \leftarrow W_{emb} \cdot Q$，截取前 $d'$ 列
3. 对每层：
   - 设 `attn_shortcut_Q = Q^T`（用于 residual connection 旋转）
   - 旋转 + 切片 Q/K/V 投影（输入维度）
   - 新的 PCA → 旋转 + 切片 O 投影（输出维度）
   - 同理处理 MLP
4. 旋转 + 切片 LM Head

**`final_orientation`**：默认 `random`，对保留维度施加随机正交变换避免 PCA 方向偏差。

## 5. Model Adapter 模式

通过 `LayerAdapter` 和 `ModelAdapter` 抽象，支持不同架构：
- **sequential**（LLaMA）：attention 和 MLP 串行，需要两个 shortcut 矩阵
- **parallel**（Phi-2）：attention 和 MLP 并行，只需一个 shortcut 矩阵

`CompressedLlamaDecoderLayer` 在 residual connection 处插入 shortcut 矩阵旋转：
```python
rotated_residual = matmul(residual, self.attn_shortcut_Q)
hidden_states = rotated_residual + hidden_states
```

## 6. 关键特点

- **稀疏性来源**：信号协方差矩阵的特征值快速衰减 → 低能量维度可安全丢弃
- **sparsity=0.25** 意味着保留 75% 维度（如 4096→3072）
- **产出稠密模型**：不同于非结构化剪枝产生稀疏矩阵，SliceGPT 产出更小但完全稠密的权重
- **无需校准数据做训练**：PCA 仅需前向推理收集协方差

## 7. 支持的模型

LLaMA-2/3、OPT、Phi-2、Phi-3
